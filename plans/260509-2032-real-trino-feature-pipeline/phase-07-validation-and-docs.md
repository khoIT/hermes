# Phase 07: Validation & Docs

## Overview

End-to-end smoke test of the new pipeline (Trino crawl ŌåÆ Postgres ŌåÆ API ŌåÆ UI), measure deltas vs. the static path, write a per-feature provenance audit, and update docs (`docs/system-architecture.md`, `docs/deployment-guide.md`, `docs/project-roadmap.md`, `infra/trino-crawler/schema-audit.md`, README) to reflect the new live wiring path.

## Requirements

**Functional:**
- A new validation script `scripts/validate-feature-pipeline.cjs` that:
  - Asserts row counts in `raw_event_aggregates`, `feature_values`, `feature_distributions_daily`, `feature_analytics_180d`.
  - Picks 5 representative features (1 per tier T1/T2/T3/T4/T5) and dumps their analytics rows + distribution rows to a JSON report.
  - Compares each feature's analytics to the prior synth fixture (`feature-analytics-180d.json`) and reports % drift on `usageCount180d`, `nullRate`, `distinctValuesP50`.
- Provenance audit: `plans/260509-2032-real-trino-feature-pipeline/reports/feature-provenance-audit.md` lists all 73 features with per-feature `source` (real / hybrid / synth) and the source-table used.
- Docs updated.

**Non-functional:**
- Validation script idempotent and re-runnable.
- Report must be readable by a non-engineer (PM-facing).

## Risk Assessment

- **VPN session expires mid-validation:** report shows partial data. **Mitigation:** validator detects 'real' source on <40 features and warns instead of asserting; user re-runs after refresh.
- **Visual regression on detail page panels:** real distributions may look different. **Mitigation:** acceptance is "no broken rendering" not "pixel-identical" ŌĆö distribution shape change is expected.
- **Doc rot:** the wiring path may evolve before the plan ships. **Mitigation:** doc updates land in the same commit as the final code; reviewer flags any references to old paths.
